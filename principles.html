第一章

动画基本原理与分类体系

三大动画范式、缓动函数与贝塞尔曲线、迪士尼 12 原则在 UI 中的映射、60fps 性能基础。

1.1 三大动画范式

帧动画（Frame Animation）

逐帧播放预定义图片序列。适用于 Lottie/GIF 加载动画、表情包等。

  • 实现方式：Android AnimationDrawable、iOS UIImage 序列、Web Canvas
  • 优点：视觉表现力极强
  • 缺点：资源占用大、不可交互、缩放模糊
  • 现代替代：Lottie（基于 Bodymovin 导出 AE 动画为矢量 JSON）

属性动画（Property Animation）

通过起始值和终止值之间插值驱动属性变化，是 UI 动画核心范式。

  • Android：ObjectAnimator、ValueAnimator
  • iOS：UIView.animate(withDuration:)、Core Animation
  • Web：CSS Transitions/Animations、Web Animations API
  • Flutter：Tween + AnimationController
  • React Native：Animated API、Reanimated
value(t) = lerp(start, end, easing(t / duration))

物理动画（Physics-based Animation）

基于物理模拟（弹簧、阻尼、重力）驱动，终点由物理状态自然收敛。

  • 关键参数：damping（阻尼比）、stiffness（刚度）、mass（质量）、velocity（初始速度）
  • 优势：动画自然有机，可中断并平滑过渡
  • 经典实现：iOS CASpringAnimation、Android SpringAnimation、Facebook POP

1.2 缓动函数与贝塞尔曲线

三次贝塞尔曲线

CSS cubic-bezier(x1, y1, x2, y2) 定义控制点 P1、P2，P0=(0,0)，P3=(1,1)。

/* 常用预设 */
ease          = cubic-bezier(0.25, 0.1, 0.25, 1.0)   /* CSS 默认 */
ease-in       = cubic-bezier(0.42, 0.0, 1.0,  1.0)   /* 慢起快停 */
ease-out      = cubic-bezier(0.0,  0.0, 0.58, 1.0)   /* 快起慢停（UI 最常用） */
ease-in-out   = cubic-bezier(0.42, 0.0, 0.58, 1.0)   /* 慢起慢停 */
Material标准  = cubic-bezier(0.4, 0.0, 0.2, 1.0)      /* Google 推荐 */
iOS 标准曲线  ≈ cubic-bezier(0.25, 0.1, 0.25, 1.0)   /* 接近 ease */

实践建议：UI 元素进场用 ease-out（快入缓停），退场用 ease-in（缓起快退），进出场景用 ease-in-out。

弹簧动画参数

参数含义典型值
stiffness（刚度 k）弹簧硬度200–1000
damping（阻尼比 ζ）能量衰减速率0.5–1.0（欠阻尼）；>1.0（过阻尼）
mass（质量 m）影响惯性通常 1.0

iOS 推荐阻尼比 0.7–0.9。

常用缓动曲线速查

名称曲线特征UI 场景
easeOut快速启动，缓慢结束元素入场、展开面板
easeIn缓慢启动，快速结束元素退场、折叠
easeOutBack超出目标再回弹按钮 pop、强调效果
easeOutElastic弹性振荡收敛游戏 UI、趣味交互
easeOutBounce弹跳效果通知弹出、下拉刷新

参考：easings.net — 缓动函数可视化速查站 · cubic-bezier.com — 在线调试器

时长黄金法则（Material Design）：小型组件 100–200ms，中型变化 200–400ms，大型场景 400–700ms。超过 700ms 用户会感知为"慢"。

1.5 60fps 与性能

  • 60fps = 每帧 16.67ms 预算；120fps = 8.33ms
  • 核心原则：只动画 transform 和 opacity（GPU 合成，不触发布局/绘制）
  • 避免动画：width/height/margin/top/left（触发 Layout → Paint → Composite）
  • 硬件加速：CSS will-change、iOS shouldRasterize、Android 默认开启
  • Web：用 requestAnimationFrame 而非 setTimeout/setInterval
  • 避免布局抖动：批量读取布局属性，批量写入
