1. 无障碍（Accessibility）与动效

prefers-reduced-motion

平台API行为
Web / RN Web@media (prefers-reduced-motion: reduce)用户开启系统"减少动态效果"后匹配
iOSUIAccessibility.isReduceMotionEnabled设置 → 辅助功能 → 减弱动态效果
AndroidAccessibilityManager.isReduceMotionEnabled (API 31+)开发者选项 → 动画缩放设为 0
RN (Reanimated)目前无内置 API，需通过 Native Module 桥接—

实践策略

// Web / RN Web
const prefersReducedMotion = typeof window !== 'undefined'
  ? window.matchMedia('(prefers-reduced-motion: reduce)').matches
  : false;

// 动画降级策略
const duration = prefersReducedMotion ? 0 : 300;
const scale = prefersReducedMotion ? 1 : withSpring(1.05);
/* CSS 降级 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

WCAG 与动效相关指南

  • WCAG 2.3.1：页面不应包含每秒闪烁超过 3 次的内容
  • WCAG 2.2 新增 2.3.2：非功能性动画应可暂停/关闭
  • 建议：所有装饰性动画提供关闭入口；功能性动画降级为瞬间切换

2. Motion Token 体系

Token 层级

Foundation（基础）→ Semantic（语义）→ Component（组件）

Foundation：duration.scale.fast / easing.standard
Semantic：motion.enter / motion.exit / motion.emphasis
Component：button.press.scale / card.expand.duration

Foundation 示例

// tokens/motion.ts
export const motion = {
  duration: {
    instant: 0,        // reduced-motion 降级
    fast: 100,         // 微交互（按钮反馈）
    normal: 200,       // 标准过渡
    moderate: 300,     // 页面切换
    slow: 500,         // 复杂场景
  },
  easing: {
    standard: [0.25, 0.1, 0.25, 1],
    decelerate: [0.0, 0.0, 0.2, 1],
    accelerate: [0.4, 0.0, 1, 1],
    emphasis: { damping: 12, stiffness: 100 },
  },
  spring: {
    gentle: { damping: 20, stiffness: 80 },
    bouncy: { damping: 10, stiffness: 150 },
    stiff: { damping: 30, stiffness: 300 },
  },
} as const;

3. 动效 A/B 测试方法论

可测试的动效指标

指标定义工具
任务完成率用户完成目标操作的比例Analytics 埋点
任务完成时间从进入页面到完成操作的时长时间戳差值
错误率因动效导致的误操作次数行为追踪
感知性能用户主观评分用户调研 / NPS
转化率动效影响的核心业务指标A/B 平台
帧率客观性能指标Perf Monitor / Flipper

A/B 测试框架

interface AnimationVariant {
  name: string;
  duration: number;
  easing: number[];
  springConfig?: { damping: number; stiffness: number };
}

const variants: Record<string, AnimationVariant> = {
  control: { name: 'control', duration: 300, easing: [0.25, 0.1, 0.25, 1] },
  faster: { name: 'faster', duration: 150, easing: [0.25, 0.1, 0.25, 1] },
  springy: { name: 'springy', duration: 0, easing: [], springConfig: { damping: 10, stiffness: 150 } },
};

const variant = await getABTestVariant('onboarding_animation');
const config = variants[variant];

注意事项

  • 动效 A/B 测试需要足够样本量（建议 ≥5000/组）
  • 区分功能动效和装饰动效
  • 注意新奇效应：新动效初期表现好可能只是因为新鲜感

4. Flutter 动效生态简要对比

维度React Native (Reanimated)Flutter
动画核心Reanimated (Worklet/SharedValue)AnimationController + Tween
隐式动画Moti / Layout presetsAnimatedContainer 等内置 Widget
弹簧动画withSpringSpringSimulation
手势+动画react-native-gesture-handler内置 GestureDetector
Hero 过渡SharedTransitionHero widget（更成熟）
自定义绘制react-native-skiaCustomPainter（内置）

Flutter 优势：内置隐式动画 Widget 开箱即用、CustomPainter 无需额外依赖、60fps 更容易保证、Hero 过渡更成熟。

Flutter 不足：设计工具→代码链路不如 Lottie 成熟、社区动效资源较少。

5. Web 与移动端动效差异

维度Web移动端
输入方式鼠标 hover/click 为主触摸手势为主
GPU 合成浏览器管理，will-change 间接控制直接控制 CALayer/RenderNode
帧率保证受浏览器标签页后台节流影响前台 App 有更高优先级
触觉反馈无Haptic Feedback
滚动性能scroll 事件在主线程原生滚动由系统处理

6. 触觉反馈（Haptics）

平台API类型
iOSUIImpactFeedbackGenerator轻触/中击/重击/成功/警告/错误
AndroidHapticFeedbackConstants点击/长按/纹理
RNexpo-haptics封装原生 API

配合原则：触觉反馈应与视觉动效同时触发（< 5ms 偏差），在动效的关键帧施加触觉信号。
