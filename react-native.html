5.2 Reanimated 核心

架构：JS 线程 vs UI 线程

JS 线程（React 渲染/业务逻辑） ⇄ SharedValue UI 线程（动画计算/手势/样式更新）

Reanimated 将动画逻辑编译为 Worklet 运行在 UI 线程。即使 JS 线程忙，动画依然 60fps。

核心 API

// SharedValue — 跨线程共享的响应式值
const x = useSharedValue(0);

// AnimatedStyle — 绑定到样式
const style = useAnimatedStyle(() => ({
  transform: [{ translateX: x.value }],
}));

// 动画函数
x.value = withTiming(100, { duration: 300, easing: Easing.bezier(0.25, 0.1, 0.25, 1) });
x.value = withSpring(100, { damping: 10, stiffness: 100, mass: 1 });
x.value = withDecay({ velocity: gestureVelocity, deceleration: 0.998 });
x.value = withSequence(withTiming(50), withSpring(100));
x.value = withRepeat(withTiming(1, { duration: 1000 }), -1, true); // 无限循环

5.3 手势动画（最核心的实战模式）

import { Gesture, GestureDetector } from 'react-native-gesture-handler';
import { useSharedValue, useAnimatedStyle, withSpring, runOnJS } from 'react-native-reanimated';

function DraggableCard({ onSwipe }) {
  const translateX = useSharedValue(0);
  const translateY = useSharedValue(0);

  const pan = Gesture.Pan()
    .onUpdate((e) => {
      translateX.value = e.translationX;
      translateY.value = e.translationY;
    })
    .onEnd(() => {
      if (Math.abs(translateX.value) > 120) {
        translateX.value = withTiming(translateX.value > 0 ? SCREEN_WIDTH : -SCREEN_WIDTH);
        runOnJS(onSwipe)(translateX.value > 0 ? 'right' : 'left');
      } else {
        translateX.value = withSpring(0);
        translateY.value = withSpring(0);
      }
    });

  const style = useAnimatedStyle(() => ({
    transform: [{ translateX: translateX.value }, { translateY: translateY.value }],
  }));

  return (
    <GestureDetector gesture={pan}>
      <Animated.View style={[styles.card, style]} />
    </GestureDetector>
  );
}

5.5 共享元素过渡（Hero Animation）

import { SharedTransition, withSpring } from 'react-native-reanimated';

const transition = SharedTransition.custom((values) => {
  'worklet';
  return {
    height: withSpring(values.targetHeight),
    width: withSpring(values.targetWidth),
    originX: withSpring(values.targetOriginX),
    originY: withSpring(values.targetOriginY),
  };
});

// 列表页
<Animated.Image sharedTransitionTag={`photo-${id}`} sharedTransitionStyle={transition} />

// 详情页
<Animated.Image sharedTransitionTag={`photo-${id}`} sharedTransitionStyle={transition} />

要求：React Navigation 6+ + react-native-screens，两端 tag 必须一致。

5.7 RN vs 原生对比

性能

维度RN (Reanimated)iOS (SwiftUI)Android (Compose)
60fps✅✅✅
120fps⚠️ 受调度限制✅⚠️
复杂粒子❌✅ (Metal)✅ (Vulkan)
共享元素✅✅✅
手势联动✅ 丝滑✅✅

何时够用 vs 何时写原生

RN 足以应对（90% 需求）：页面过渡、列表动画、手势交互动画、Bottom Sheet、Lottie、布局动画

需原生模块：高性能粒子系统、3D 动画、视频实时滤镜、自定义导航容器、120fps 精确控制

5.8 性能最佳实践

  1. 永远用 SharedValue，不要用 state 驱动动画（避免每帧 re-render）
  2. 避免 Worklet 中的重计算（会阻塞 UI 线程）
  3. 列表动画用 CellRendererComponent
  4. 及时清理动画（cancelAnimation）
  5. 用 runOnJS 桥接 JS 函数调用（不要在 UI 线程直接调 JS 函数）
