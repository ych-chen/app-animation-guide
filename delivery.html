动效交付流程与工程化

Motion Spec 参数规范、导出格式对比、交付流程、各平台代码接入、性能优化。

4.1 Motion Spec 参数规范

一个完整的动效 Spec 应包含：

参数类别字段示例值
基础name / trigger / duration / delaycard_expand / onTap / 300ms
缓动easing / spring_paramscubic-bezier(0.25, 0.1, 0.25, 1)
运动property / from / to / pathtransform.scale / 0.8 → 1.0
视觉opacity / blur / shadow0 → 1 / 10px → 0px
交互interruptible / repeat / gesture_driventrue / none / false

YAML 模板示例

animation:
  name: "card_expand"
  trigger: "onTap"
  duration: 300
  properties:
    - property: "transform.scale"
      from: 0.95
      to: 1.0
      easing: "cubic-bezier(0.4, 0.0, 0.2, 1)"
    - property: "opacity"
      from: 0
      to: 1
      easing: "cubic-bezier(0.0, 0.0, 0.2, 1)"
  interaction:
    interruptible: true

4.2 导出格式对比

格式透明大小RN 支持交互推荐场景
Lottie JSON✅极小(5-100KB)✅高图标/加载/插画 首选
Rive✅极小✅极高可交互 UI/游戏
GIF❌大✅无表情包
MP4/WebM❌中✅低开屏/品牌视频
APNG✅大⚠️无表情贴纸
SVG 动画✅极小❌中Web 专用
WebP 动画✅较小⚠️无Web 替代 GIF
PNG 序列帧✅极大✅低最后 fallback

决策树：矢量/简单动画 → Lottie ✅ | 需要交互/状态机 → Rive ✅ | 视频级特效 → MP4 | Web 专用 → SVG/Lottie Web

4.3 交付流程

Phase 1: 设计 → AE 原型→ Phase 2: 交付 → Lottie/Spec→ Phase 3: 开发 → 代码实现→ Phase 4: 验收 → 逐帧对比

验收标准：时长匹配±10%、缓动曲线匹配、属性值匹配±5%、60fps 无掉帧、无内存泄漏。

4.4 各平台代码接入

React Native — Lottie

npm install lottie-react-native lottie-ios
cd ios && pod install
<LottieView
  source={require('./animations/loading.json')}
  autoPlay loop
  colorFilters={[{ keypath: 'layer', color: '#FF6B6B' }]}
/>

React Native — Reanimated

import Animated, { useSharedValue, useAnimatedStyle, withSpring } from 'react-native-reanimated';

const scale = useSharedValue(1);
const style = useAnimatedStyle(() => ({ transform: [{ scale: scale.value }] }));
scale.value = withSpring(0.95, { damping: 15, stiffness: 400 });

iOS 原生

// UIKit Spring
UIView.animate(withDuration: 0.5, usingSpringWithDamping: 0.7,
  initialSpringVelocity: 0.5, options: .curveEaseInOut) {
    self.cardView.frame.origin.y = targetY
}

// SwiftUI
RoundedRectangle(cornerRadius: 16)
    .scaleEffect(isExpanded ? 1.0 : 0.95)
    .animation(.spring(response: 0.4, dampingFraction: 0.7), value: isExpanded)

Android 原生

// ViewPropertyAnimator
cardView.animate().alpha(1f).scaleX(1f).setDuration(300).start()

// Compose
val scale by animateFloatAsState(
    targetValue = if (expanded) 1f else 0.95f,
    animationSpec = spring(dampingRatio = Spring.DampingRatioMediumBouncy)
)

4.5 性能优化

通用原则

  • 仅动画 transform 和 opacity
  • 避免布局抖动（Layout Thrashing）
  • 控制并发动画数（同屏 ≤ 3 个复杂动画）
  • RN 中动画逻辑运行在 UI 线程（Reanimated）
  • 及时清理：动画结束后移除 listener、释放资源
  • 预加载 Lottie JSON

性能预算

指标目标
动画帧率≥ 60fps
单帧耗时≤ 16ms（60fps）/ ≤ 8ms（120fps）
Lottie JSON 大小≤ 100KB
动画内存增量≤ 5MB
并发 Lottie 动画≤ 3 个同屏
