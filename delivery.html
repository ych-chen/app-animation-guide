动效交付流程与工程化

Motion Spec 参数规范、导出格式对比、交付流程、各平台代码接入、性能优化。

4.1 Motion Spec 参数规范

一个完整的动效 Spec 应包含：

参数类别字段示例值
基础name / trigger / duration / delaycard_expand / onTap / 300ms
缓动easing / spring_paramscubic-bezier(0.25, 0.1, 0.25, 1)
运动property / from / to / pathtransform.scale / 0.8 → 1.0
视觉opacity / blur / shadow0 → 1 / 10px → 0px
交互interruptible / repeat / gesture_driventrue / none / false

YAML 模板示例

animation:
  name: "card_expand"
  trigger: "onTap"
  duration: 300
  properties:
    - property: "transform.scale"
      from: 0.95
      to: 1.0
      easing: "cubic-bezier(0.4, 0.0, 0.2, 1)"
    - property: "opacity"
      from: 0
      to: 1
      easing: "cubic-bezier(0.0, 0.0, 0.2, 1)"
  interaction:
    interruptible: true

4.2 导出格式对比

格式透明大小RN 支持交互推荐场景
Lottie JSON✅极小(5-100KB)✅高图标/加载/插画 首选
Rive✅极小✅极高可交互 UI/游戏
GIF❌大✅无表情包
MP4/WebM❌中✅低开屏/品牌视频
APNG✅大⚠️无表情贴纸
SVG 动画✅极小❌中Web 专用
WebP 动画✅较小⚠️无Web 替代 GIF
PNG 序列帧✅极大✅低最后 fallback

决策树：矢量/简单动画 → Lottie ✅ | 需要交互/状态机 → Rive ✅ | 视频级特效 → MP4 | Web 专用 → SVG/Lottie Web

4.3 各格式交付物清单与使用说明

「光给文件是不够的」—— 每种格式都有开发者必须知道的配套信息，遗漏会导致开发反复沟通甚至无法接入。

📦 Rive (.riv)

⚠️ 最容易遗漏信息的格式。Rive 支持状态机、事件和交互，开发者需要知道所有可交互的入口。

必须交付：

  • .riv 文件本身
  • Artboard 名称 — 如果文件包含多个画板，需指明使用哪个（默认是 main，但自定义名称必须提供）
  • State Machine 名称 — 状态机名称是开发者加载动画时的必填参数
  • 所有 Input 名称及类型：
    • Boolean 输入（如 isHovered、isPressed）— 什么时候该设为 true/false
    • Number 输入（如 progress 0→1、scrollOffset）— 数值范围和含义
    • Enum 输入（如 state: "idle" | "loading" | "success" | "error"）— 所有枚举值
  • Event 监听 — 如果状态机中定义了 Rive Event，需列出事件名称和触发时机
  • 交互说明文档 — 哪些用户操作对应哪些 Input 变化

交付模板示例：

rive_delivery:
  file: "toggle_switch.riv"
  artboard: "main"
  state_machine: "toggle_state"
  inputs:
    - name: "isOn"
      type: "boolean"
      description: "开关状态，true=开，false=关"
    - name: "progress"
      type: "number"
      range: [0, 1]
      description: "拖拽进度，0=完全关闭，1=完全打开"
  events:
    - name: "toggleComplete"
      description: "开关动画播放完毕时触发"
  interaction_map:
    - user_action: "点击"
      input_change: "isOn → toggle"
    - user_action: "拖拽"
      input_change: "progress → 跟随手指位置"

RN 接入代码（配合上面信息）：

import Rive, { RiveRef } from 'rive-react-native';

const toggleRef = useRef<RiveRef>(null);

<Rive
  resourceName="toggle_switch"
  artboardName="main"          // ← 必须指定
  stateMachineName="toggle_state" // ← 必须指定
  ref={toggleRef}
/>

// 触发交互：
toggleRef.current?.setInputState("toggle_state", "isOn", true);
toggleRef.current?.fireState("toggle_state", "toggleComplete");

📦 Lottie JSON (.json)

必须交付：

  • .json 文件（或 .lottie 压缩包）
  • 需要动态着色的图层路径（Keypath）— 如果需要主题色适配，列出哪些图层名需要换色
    • 示例："Shape Layer 1.Group 1.Fill 1" → 替换为主题色 #FF6B6B
  • Markers（标记点） — 如果一个 JSON 包含多段动画（如 idle、loading、success），需列出各段名称和起止帧
  • 循环说明 — 哪些段循环播放、哪些播一次
  • 图片资源 — 如果 JSON 引用了外部图片，需同时交付 images/ 文件夹

交付模板示例：

lottie_delivery:
  file: "loader.json"
  loop: true
  speed: 1.0
  markers:
    - name: "spin"
      frames: [0, 60]
      loop: true
    - name: "success"
      frames: [61, 90]
      loop: false
  color_replacements:
    - keypath: "Shape Layer 1.Group 1.Fill 1"
      default_color: "#3B82F6"
      description: "主品牌色"
    - keypath: "Shape Layer 2.Group 1.Stroke 1"
      default_color: "#93C5FD"
      description: "辅助色"
  external_images: false  # true 时需附 images/ 目录

📦 MP4 / WebM 视频

必须交付：

  • 视频文件（建议同时提供 MP4 H.264 + WebM VP9 两种格式）
  • 分辨率（如 1080×1920）— 是否需要 @2x/@3x
  • 帧率（通常 30fps 或 60fps）
  • 循环点 — 如果需要无缝循环，指明循环点的时间码
  • 透明通道 — MP4 不支持透明，如果需要透明背景，说明使用 WebM（VP9 + alpha）或提供遮罩视频
  • 播放控制 — 是否需要预加载、是否自动播放、播完后的行为

📦 APNG / WebP 动画

必须交付：

  • 文件本身
  • 帧率（APNG 默认可能不是你期望的）
  • 播放次数（无限循环 / 播放 N 次）
  • 平台兼容性说明：
    • APNG：iOS 原生支持，Android 9+ 支持，旧版需用库（如 apng-drawable）
    • WebP 动画：Android 4.0+ 支持，iOS 需额外处理

📦 PNG 序列帧

必须交付：

  • 所有帧图片，按命名规则排列（如 frame_0001.png ~ frame_0060.png）
  • 帧率（如 30fps = 60帧播放2秒）
  • 单帧尺寸（如 200×200px）— 是否需要 @2x/@3x
  • 命名规则 — 确保开发者能自动解析排序
  • 是否需要透明背景
  • 总文件大小 — 如果过大，需要讨论是否压缩或换格式

📦 SVG 动画

必须交付：

  • SVG 文件（内含 CSS/SMIL 动画）
  • 动画触发方式 — 页面加载自动播放 / 滚动触发 / 点击触发
  • 浏览器兼容性：
    • SMIL 动画：Chrome/Firefox/Safari 支持，IE/旧 Edge 不支持
    • CSS 动画：兼容性较好
    • 如需广泛兼容，建议改用 Lottie Web

📦 GIF

必须交付：

  • GIF 文件
  • ⚠️ 不支持透明通道（只有全透明或全不透明，没有半透明）
  • 文件大小 — GIF 通常很大，需要确认是否可接受
  • 分辨率 — 建议控制在 200px 以内

💡 通用规则：任何格式交付时，都应该附带一份 Markdown/YAML 说明文档，包含：①文件清单 ②关键参数（帧率/分辨率/循环方式） ③交互映射（如果是交互式动画） ④已知限制。把这份文档和文件一起放在交付文件夹中，能减少 80% 的来回沟通。

4.4 交付流程

Phase 1: 设计 → AE 原型→ Phase 2: 交付 → Lottie/Spec→ Phase 3: 开发 → 代码实现→ Phase 4: 验收 → 逐帧对比

验收标准：时长匹配±10%、缓动曲线匹配、属性值匹配±5%、60fps 无掉帧、无内存泄漏。

React Native — Reanimated

import Animated, { useSharedValue, useAnimatedStyle, withSpring } from 'react-native-reanimated';

const scale = useSharedValue(1);
const style = useAnimatedStyle(() => ({ transform: [{ scale: scale.value }] }));
scale.value = withSpring(0.95, { damping: 15, stiffness: 400 });

// UIKit Spring
UIView.animate(withDuration: 0.5, usingSpringWithDamping: 0.7,

4.6 性能优化

通用原则

  • 仅动画 transform 和 opacity
  • 避免布局抖动（Layout Thrashing）
  • 控制并发动画数（同屏 ≤ 3 个复杂动画）
  • RN 中动画逻辑运行在 UI 线程（Reanimated）
  • 及时清理：动画结束后移除 listener、释放资源
  • 预加载 Lottie JSON

性能预算

指标目标
动画帧率≥ 60fps
单帧耗时≤ 16ms（60fps）/ ≤ 8ms（120fps）
Lottie JSON 大小≤ 100KB
动画内存增量≤ 5MB
并发 Lottie 动画≤ 3 个同屏
