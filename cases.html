优秀 App 动效案例

Apple Music、Discord、Pinterest 等标杆案例深度分析，GitHub 仓库推荐，从设计到代码的完整链路。

3.1 标杆案例

Apple Music（iOS 原生）

项目内容
动效类型共享元素过渡、全屏播放器展开/收起、专辑封面旋转
实现方式SwiftUI matchedGeometryEffect + UIViewPropertyAnimator
亮点从迷你播放器到全屏播放器的无缝过渡，封面平滑变形，可交互式中断
// SwiftUI 共享过渡核心代码
@Namespace var albumArtNamespace
Image("album")
    .matchedGeometryEffect(id: "albumArt", in: albumArtNamespace)

Discord（React Native）⭐ RN 标杆

项目内容
动效类型侧滑抽屉、消息列表滚动、底部 Sheet、shimmer 效果
实现方式react-native-reanimated + react-native-gesture-handler，动画全部运行在 UI 线程
亮点RN 性能优化标杆，60fps 复杂交互，证明 RN 可以做出原生级动效
// Discord 典型模式：手势 + 弹簧动画
const panGesture = Gesture.Pan()
  .onUpdate((e) => { translateX.value = e.translationX; })
  .onEnd(() => {
    translateX.value = withSpring(0); // 或 withSpring(-MAX_WIDTH)
  });

详见：Discord RN 性能博客

Pinterest（React Native）

项目内容
动效类型瀑布流列表动画、Pin 详情共享元素过渡、保存按钮动画
实现方式Reanimated staggered animation + 自定义 Shared Transition
亮点瀑布流布局+丰富动画从不卡顿

Google Wallet（Android 原生）

项目内容
动效类型卡片翻转、NFC 支付涟漪、卡片堆叠选择
实现方式Jetpack Compose + Material Design 3 Motion
亮点MD3 Motion 教科书实现，动效服务于功能确认

3.2 从设计到代码的完整链路

Figma/AE 设计→ 对齐参数→ 选择实现方式→ 编码+调试
  1. 设计师在 Figma/AE 中制作动效原型
  2. 对齐参数：缓动曲线、时长、延迟
  3. 选择实现方式：
    • 简单过渡 → Reanimated withTiming/withSpring
    • 复杂矢量动画 → Lottie JSON
    • 交互动画 → Reanimated + Gesture Handler
  4. 编码实现 → 真机对比调试 → 性能测试 → 迭代修改

关键实践：将时长、缓动曲线、延迟提取为常量集中管理，方便调整。
